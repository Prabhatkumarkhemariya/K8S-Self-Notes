What is a Shell Script?
A shell script is simply a text file that contains a series of commands written for a shell (like bash, sh, zsh, etc.) to execute — automatically and in sequence.

Think of it like this:
A shell script = a batch of Linux commands you write once → then the computer runs them one by one automatically.

What is “shell”?
A shell is a program that takes commands from you and gives them to the operating system to execute.
Examples:
| Shell Name | Command Interpreter | Commonly Used In         |
| ---------- | ------------------- | ------------------------ |
| `bash`     | Bourne Again Shell  | Linux / Git Bash / macOS |
| `sh`       | Bourne Shell        | Legacy systems           |
| `zsh`      | Z Shell             | macOS, advanced users    |
| `ksh`      | Korn Shell          | Unix systems             |
So a shell script is written in a specific shell’s language — most commonly Bash (#!/bin/bash).

=> What 2>/dev/null Means
In shell (Bash), every process has three standard data streams:
| Stream     | Number | Purpose              | Example                           |
| ---------- | ------ | -------------------- | --------------------------------- |
| **stdin**  | `0`    | Input to the program | Keyboard, file input              |
| **stdout** | `1`    | Normal output        | Command results                   |
| **stderr** | `2`    | Error messages       | “No such file or directory”, etc. |

>/dev/null
Means “send standard output (1)** to nowhere**.”
So if you write:
command > /dev/null
you’re hiding all normal output (but errors still show).

2>/dev/null
Means “send standard error (2)** to nowhere**.”
So if you write:
command 2>/dev/null
you’re hiding all error messages but keeping normal output visible.
/dev/null
It’s a special file in Linux that acts like a black hole
→ Anything written to it just disappears.

=> find "$dir" -type f -name "*.log" -mtime +"$days" -exec rm -f {} \;
Step-by-Step Explanation
| Part                | Meaning                                                                                                                   |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `find`              | The Linux command used to **search for files/directories** based on conditions (name, type, time, size, etc.)             |
| `"$dir"`            | The **directory path** where the search starts. It’s a variable (e.g., `/var/logs` or `/d/DevOps_Interview_question/...`) |
| `-type f`           | Look only for **files** (not directories, links, etc.)                                                                    |
| `-name "*.log"`     | Match only files ending with `.log`                                                                                       |
| `-mtime +"$days"`   | Match files last modified more than `$days` days ago                                                                  |
| `-exec rm -f {} \;` | For each file found (`{}` = placeholder for the file path), execute the command `rm -f` (force delete it)                 |

`-mtime +"$days"
| Part      | Meaning                                                                                |
| --------- | -------------------------------------------------------------------------------------- |
| `-mtime`  | **Modification time** — how long ago the file was last modified (in *days*).           |
| `+`       | Means **“greater than”**.                                                              |
| `"$days"` | A **variable** that stores the number of days you want to compare (e.g., 7, 10, etc.). |

| Expression  | Meaning                             | Example (if `$days` = 7) |
| ----------- | ----------------------------------- | ------------------------ |
| `-mtime +7` | Modified **more than 7 days ago**   | older than 7 days        |
| `-mtime 7`  | Modified **exactly 7 days ago**     | between 7 and 8 days ago |
| `-mtime -7` | Modified **within the last 7 days** | newer than 7 days        |

-exec rm -f {} \;
| Part    | Meaning                                                                                  |
| ------- | ---------------------------------------------------------------------------------------- |
| `-exec` | Tells `find` to **execute a command** on each file it finds                              |
| `rm -f` | The command to **forcefully remove (delete)** the file                                   |
| `{}`    | A **placeholder** for the current file found by `find`                                   |
| `\;`    | Marks the **end of the command**; required to tell `find` where the `-exec` command ends |

The \; tells find: “This is the end of the command you want me to execute.”
Without \;, find would keep waiting for more input and show an error.
rm -f {}   # deletes the file found by find

| Part       | ------------------------Meaning--------------------------|

Why i=5?

[Date] [Time] [Server] [App]: [Message Words…]

=> tar -tzf "$backup_file" >/dev/null 2>&1
meaning of >/dev/null 2>&1 both
First, understand outputs in Linux:
Every Linux command can produce two types of output:
| Stream | Name       | Description    | Example              |
| ------ | ---------- | -------------- | -------------------- |
| `1`    | **stdout** | Normal output  | “Backup successful!” |
| `2`    | **stderr** | Error messages | “File not found!”    |
Part 1: >/dev/null
This means:“Send stdout (1) to /dev/null.” : /dev/null is a special black hole file — anything written there disappears forever.
But if an error occurs, it still appears on your screen because errors (stderr) are separate.
Part 2: 2>&1:
This means:“Send stderr (2) to the same place as stdout (1).”
Now both normal output and errors go to the same place.
Since stdout was already sent to /dev/null,
stderr also gets sent there.
&>/dev/null = >/dev/null 2>&1
Summary Table:
| Command           | Meaning                     | Works in   | Description         |
| ----------------- | --------------------------- | ---------- | ------------------- |
| `>/dev/null`      | Hide stdout only            | all shells | Normal output muted |
| `2>/dev/null`     | Hide stderr only            | all shells | Only errors muted   |
| `>/dev/null 2>&1` | Hide both stdout and stderr | all shells | Traditional method  |
| `&>/dev/null`     | Hide both stdout and stderr | Bash & Zsh | Shorter syntax      |
